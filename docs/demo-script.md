# 星露谷小助手演示脚本

这份脚本用来快速检查项目是否像一个“能用的 AI 游戏小助手”，而不是只会展示资料卡片。

## 演示前准备

1. 启动前端：

```bash
python -m http.server 4173 --bind 127.0.0.1
```

2. 启动后端：

```bash
python -m uvicorn backend.app.main:app --host 127.0.0.1 --port 8000
```

3. 打开：

```text
http://127.0.0.1:4173
```

## 推荐演示路径

### 1. 闲聊边界

问题：

```text
你好
```

期望：

- 不要乱检索草莓、作物或社区中心。
- 应该自然介绍自己是星露谷小助手。

### 2. 精准资料命中

问题：

```text
蛋节草莓怎么买最划算？
```

期望：

- 第一参考资料应是 `Egg Festival Strawberry 蛋节草莓购买`。
- 回答应提到春 13 蛋节、买完立刻种、按浇水压力控制数量。
- 回答下方应出现参考资料、继续追问和“查看诊断”。

### 3. 玩家状态影响回答

先切换玩家状态：

```text
预算紧 + 手浇
```

再问：

```text
草莓买多少合适？
```

期望：

- 回答应提醒不要买到每天浇水崩溃。
- 应给出偏保守的数量建议。

### 4. 中期路线问题

切换预设：

```text
中期赚钱
```

问题：

```text
啤酒花怎么赚钱？
```

期望：

- 应命中啤酒花/淡啤酒/小桶路线。
- 回答要像路线建议，不只是百科定义。

### 5. 便签工作流

在任意一条有用回答下点击：

```text
加入便签
```

期望：

- 自动切到“维护”面板。
- “路线便签”下出现这条回答摘要。
- 出现“复制”和“清空”按钮。
- 点击“复制”后 toast 显示 `路线便签已复制`。

### 6. 资料缺口保护

问题：

```text
祝尼魔赛车怎么过？
```

期望：

- 如果资料库没有足够内容，不应硬扯到无关作物或礼物。
- 应提示资料不足，并可在诊断里看到待补资料建议。
- 点击 `生成草稿` 后，会先加入 backlog，再在对应 Markdown 文件中生成低可信草稿。

### 7. 新增高频资料

问题：

```text
齐先生任务怎么做？
鲶鱼怎么钓？
鱼塘值不值得建？
农舍什么时候升级？
```

期望：

- 齐先生问题应命中 `Mr Qi Special Orders 齐先生任务`。
- 鲶鱼问题应提醒春/秋雨天和鱼缸留样。
- 鱼塘问题应说明鲟鱼、鱼籽、鱼子酱和中期优先级。
- 农舍升级应说明厨房、地窖和预算取舍。

## 回归检查

修改检索或资料库后跑：

```bash
node --check app.js
python -m compileall backend scripts
python scripts/test-retrieval.py
```

这三条都通过，再打开浏览器试一次“蛋节草莓怎么买最划算？”。
